# Contributing to Campfire

Thank you for your interest in contributing to Campfire! This guide covers everything you need to get started with development.

## Prerequisites

- [Bun](https://bun.sh) >= 1.0
- [Git](https://git-scm.com/)
- At least one agent CLI installed for testing (e.g., [Claude Code](https://docs.anthropic.com/en/docs/claude-code))

## Setup

```bash
# Clone the repo
git clone https://github.com/stretchcloud/campfire.git
cd campfire/web

# Install dependencies
bun install
```

## Development Mode

Development mode gives you hot module reloading (HMR) for frontend changes ‚Äî the browser updates instantly without a full reload.

```bash
bun run dev
```

This starts two processes internally:
- **Backend** on `http://localhost:14567` (auto-restarts on file changes)
- **Frontend** on `http://localhost:4567` (Vite HMR, proxies API/WS to backend)

Open **http://localhost:4567** ‚Äî this is the only URL you need. The internal backend port (14567) is never accessed directly.

## Production Mode

```bash
bun run build    # Build frontend into dist/
bun run start    # Start production server on :4567
```

## Commands

```bash
bun run dev          # Start dev server (backend + frontend with HMR)
bun run build        # Production build
bun run start        # Start production server
bun run typecheck    # TypeScript validation
bun run test         # Run all tests
bun run test:watch   # Watch mode
```

## Code Quality

Before committing, always run:

```bash
bun run typecheck    # Must pass with zero errors
bun run test         # Tests must pass
```

CI runs the same gate on every pull request and push to `main` (see `.github/workflows/ci.yml`): typecheck, the full test suite, and a production build. All three must pass before a PR can merge. A husky pre-commit hook runs typecheck + tests locally so failures surface before you push.

### Key style rules we follow

- **No nested ternaries** ‚Äî extract helper functions
- **Readonly props** ‚Äî use `Readonly<>` on component props
- **No `String(e)`** ‚Äî use `"Unknown error"` for catch blocks
- **`globalThis` over `window`**
- **Labels associated with controls** ‚Äî use `htmlFor` + `id`
- **Keep cognitive complexity low** ‚Äî extract logic into helpers

## Testing

```bash
# Run all tests
bun run test

# Run specific test file
bun test server/webhook-manager.test.ts

# Watch mode
bun run test:watch
```

Tests live alongside source files (e.g., `routes.test.ts` next to `routes.ts`). All new code should include tests when possible.

## Writing Adapters

See [`web/server/ADAPTERS.md`](web/server/ADAPTERS.md) for a step-by-step guide on adding new agent backends. Community adapters can be published as npm packages with a `campfireAdapter` field in `package.json`.

## Architecture

```
Browser (React 19) ‚Üê‚Üí WebSocket ‚Üê‚Üí Hono Server (Bun) ‚Üê‚Üí Agent Backend
     :4567              /ws/browser/:id     :4567         (Claude/Codex/Goose/etc.)
```

- All user traffic goes through port **4567**
- In dev mode, Vite runs on 4567 and proxies API/WS to the internal backend on 14567
- In production, Hono serves both the static frontend and the API on 4567
- Agent CLIs connect back to the server via WebSocket (`/ws/cli/:id`)

## Pull Requests

- Use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages
- Run `bun run typecheck` and `bun run test` before submitting
- Explain what the PR does and why
- Add screenshots for visual changes
